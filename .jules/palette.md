## 2024-05-24 - Accessibility refactoring for interactive elements
**Learning:** In order for users navigating via keyboard and screen readers to access and interact with lists and table rows, non-semantic HTML elements like `<div onClick>` or `<td onClick>` must be refactored to native `<button>` elements. These elements also require focus indicators (like `focus:ring-2` with `outline-none`) to show they are active. Dynamic UI components like toast notifications need ARIA attributes (`role="alert"` and `aria-live="assertive"`) to ensure screen readers announce updates immediately.
**Action:** When adding clickable elements, always use `<button>` instead of semantic structural tags. Always define `role="alert"` and `aria-live="assertive"` on any popup notifications.
